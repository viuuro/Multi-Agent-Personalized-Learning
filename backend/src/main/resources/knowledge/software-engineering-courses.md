# 软件工程基础

## 学科目标与生命周期

软件工程研究如何在成本、进度、质量和风险约束下，以系统化、可重复的方法开发与演化软件。完整生命周期通常包含可行性分析、需求、设计、构造、测试、部署、运维和维护。过程并非只能顺序执行：需求较稳定且审计严格时可采用阶段化模型，变化快的产品更适合迭代与增量交付，风险高的项目应尽早用原型验证关键假设。

## 需求、设计与质量

需求工程包括获取、分析、规格说明、确认和变更管理。功能需求描述系统行为，非功能需求描述性能、安全、可靠性和可用性等可验证约束。设计应追求高内聚、低耦合、信息隐藏和清晰接口；架构决策要记录背景、方案、权衡与后果。测试需要覆盖正常、边界和异常路径，并结合需求风险解释覆盖率。

## 协作与持续交付

版本控制、代码评审、自动化测试和持续集成用于缩短反馈周期。配置管理还应覆盖依赖、数据库脚本、部署配置和制品。项目管理关注范围、时间、成本、人员和风险，估算需要表达假设与不确定性。DevOps 强调开发、测试、运维和安全协作，通过流水线、可观测性、灰度发布与回滚能力持续交付价值。

# C语言程序设计

## 程序结构与类型

C 程序由预处理指令、声明、函数和表达式组成。需要掌握基本类型、整数提升、隐式转换、作用域、存储期，以及顺序、选择和循环结构。函数声明应准确描述参数与返回值；头文件用于公开接口，源文件保存实现，避免重复定义和不受控的全局状态。

## 指针、数组与内存

数组在多数表达式中会退化为首元素指针，但数组和指针并不等价。指针运算必须位于同一数组对象范围内；字符串以空字符结尾，缓冲区需要预留终止符空间。自动存储对象随作用域结束失效，动态内存由 malloc、calloc、realloc 和 free 管理。越界、悬空指针、重复释放和未初始化读取属于典型未定义行为。

## 数据组织与工程实践

结构体组织相关数据，联合体让多个成员共享存储，枚举表达有限状态。文件和标准输入输出应检查返回值并正确处理 EOF。宏只做文本替换，复杂逻辑优先使用函数。编译时开启告警，结合调试器、断言、单元测试和内存检查工具定位问题，并把声明、实现和构建脚本清晰分层。

# C++程序设计

## 对象、资源与值语义

C++ 在 C 的基础上提供类、泛型和更强的资源管理机制。构造函数建立对象不变量，析构函数释放资源，RAII 将资源生命周期绑定到对象生命周期。需要区分值、引用、指针、左值与右值，理解拷贝和移动语义。优先用标准容器、string 和智能指针表达所有权，减少手工 new 与 delete。

## 抽象、多态与泛型

封装隐藏实现细节，继承用于表达真正的 is-a 关系，运行时多态依赖虚函数和稳定接口。基类若通过基类指针销毁派生对象，析构函数通常应为虚函数。模板支持编译期泛型，概念与约束能让接口意图更清晰。组合往往比深层继承更易维护。

## 标准库与现代实践

STL 由容器、迭代器、算法和函数对象组成，应依据访问模式和失效规则选择 vector、deque、list、map 或 unordered_map。异常用于报告无法在局部处理的失败，代码需保持基本或强异常安全保证。现代 C++ 项目还应掌握 lambda、范围、并发原语、CMake、静态分析、测试和性能剖析。

# Java程序设计

## 类型系统与面向对象

Java 源代码编译为字节码并由 JVM 执行。基本类型保存值，对象变量保存引用；理解装箱、拆箱、equals 与 hashCode 契约是使用集合的基础。类通过封装维护不变量，接口定义能力边界，继承应服务于可替换性。不可变对象通常更易推理和并发共享。

## 集合、异常与输入输出

List、Set、Map 和 Queue 面向不同的数据关系，泛型提供编译期类型安全。受检异常适合调用方可恢复的问题，运行时异常通常表示编程错误或无效状态；资源使用 try-with-resources 自动关闭。文件、网络和序列化操作要明确字符编码、缓冲策略和不可信输入风险。

## 并发、构建与测试

线程安全不仅是避免崩溃，还要保证可见性、原子性和有序性。优先使用 Executor、并发集合、Future 或结构化并发抽象，谨慎管理锁的粒度与顺序。项目通过 Maven 或 Gradle 管理依赖和生命周期，使用 JUnit、Mock 和集成测试验证不同层次，并借助日志、监控和性能分析观察 JVM 行为。

# Java EE 企业级开发

## 平台与分层架构

Java EE 已演进为 Jakarta EE，包名逐步从 javax 迁移到 jakarta。企业应用常按接口、应用服务、领域和基础设施分层，控制器负责协议适配，业务规则不应散落在页面或数据库访问代码中。Spring Boot 生态也广泛用于实现相同类型的企业服务，但要区分框架特性与平台标准。

## Web、持久化与事务

Servlet 或 REST 控制器处理 HTTP 请求、状态码、内容协商与参数校验。JPA 用实体和关系映射持久化数据，需要理解实体生命周期、懒加载、N+1 查询和级联边界。事务应围绕完整业务用例设置，明确传播、隔离、回滚条件和幂等性，不能用长事务包住远程调用。

## 安全、集成与部署

身份认证确认用户是谁，授权决定可以执行什么操作。服务应验证输入、保护密钥、防止注入和越权，并采用安全的会话或令牌策略。企业集成还涉及消息队列、缓存、任务调度和第三方服务。容器化部署需要外部化配置、健康检查、结构化日志、指标、链路追踪和可回滚发布。

# Python程序设计

## 语言基础与数据模型

Python 以缩进组织代码，常用类型包括数字、字符串、列表、元组、集合和字典。变量绑定对象而不是固定存储槽，需要理解可变与不可变对象、浅拷贝与深拷贝、可迭代对象、迭代器和生成器。推导式适合简洁转换，但复杂业务逻辑应保留可读结构。

## 函数、模块与面向对象

函数是一等对象，参数支持位置、关键字、默认值和可变参数；默认可变参数会跨调用共享，应避免使用。模块和包组织命名空间，虚拟环境与依赖锁定保证可复现。类可通过组合、继承、数据类和协议表达抽象，特殊方法定义对象如何参与迭代、比较和上下文管理。

## 工程与异步编程

异常应在能够恢复或补充上下文的边界处理，文件和连接用 with 管理。类型标注、格式化、静态检查、pytest 和日志提升大型项目可维护性。线程适合 I/O 并发，进程适合 CPU 密集任务，asyncio 通过事件循环管理大量异步 I/O；选择模型前应先识别真正瓶颈。

# 数据库系统

## 数据模型与 SQL

关系模型用表、元组、属性、键和约束描述数据。SQL 包含查询、定义、操作和权限控制，连接条件决定多表组合语义。聚合、子查询、窗口函数和公共表表达式能表达复杂分析。参数化查询用于分离代码与数据，避免 SQL 注入。

## 设计、索引与查询优化

概念设计通过实体、关系和约束描述业务，逻辑设计将其映射为关系模式。规范化用于减少冗余和更新异常，但性能场景可在测量后有控制地反规范化。B+树索引适合范围与排序，复合索引受列顺序影响；执行计划、选择率、统计信息和访问行数是优化查询的重要依据。

## 事务、恢复与安全

事务的 ACID 属性分别关注原子性、一致性、隔离性和持久性。并发可能产生脏读、不可重复读、幻读和写冲突，数据库通过锁、MVCC 和隔离级别协调。日志、检查点、备份和恢复策略共同保护数据。权限应遵循最小授权，敏感数据需要加密、脱敏、审计和生命周期治理。

# 数据结构

## 复杂度与线性结构

时间和空间复杂度描述输入规模增长时资源消耗的数量级，还需区分最坏、平均与摊还分析。数组支持常数时间随机访问，链表适合已知位置的局部插删；栈表达后进先出，队列表达先进先出，双端队列支持两端操作。选择结构应依据实际操作分布而非名称熟悉度。

## 树、散列与图

二叉搜索树保持有序关系，平衡树限制高度，堆高效支持优先级操作，Trie 适合前缀检索。散列表平均可实现常数时间查找，但需处理哈希质量、冲突、扩容与负载因子。图由顶点和边组成，邻接表适合稀疏图，邻接矩阵适合稠密图；遍历、最短路、最小生成树和拓扑排序解决不同关系问题。

## 排序、查找与解题

稳定性、原地性、比较次数和数据分布会影响排序算法选择。归并排序稳定且适合外部排序，快速排序平均性能优秀，堆排序提供稳定的复杂度上界。二分查找要求单调或有序条件，并需谨慎处理边界。解题时先定义不变量和接口，再用样例、边界和复杂度验证实现。

# 离散数学

## 逻辑与证明

命题逻辑研究联结词、真值和等价关系，谓词逻辑加入量词与个体。证明方法包括直接证明、逆否证明、反证法、分类讨论和数学归纳法。证明算法正确性时，循环不变量连接初始化、保持和终止三个环节。

## 集合、关系与计数

集合运算、笛卡尔积、函数和关系是描述软件对象与约束的基础。等价关系产生划分，偏序关系刻画先后或依赖。计数原理包括加法、乘法、排列组合、容斥和鸽巢原理；递推关系描述状态随规模变化，可用展开、特征方程或生成函数分析。

## 图论与代数结构

图用于建模网络、调用、依赖和状态迁移。连通性、路径、环、树、匹配、着色和平面性对应不同工程问题。有向无环图可进行拓扑排序。布尔代数支撑逻辑电路和条件化简，群、环、域等代数结构为密码学、编码和抽象运算提供统一语言。

# 线性代数

## 向量、矩阵与线性方程

向量表示有方向的数据或特征，矩阵表示线性变换和多组关系。矩阵加法、乘法、转置和逆各有明确维度条件。线性方程组可通过高斯消元化为阶梯形，解的存在性与唯一性由系数矩阵和增广矩阵的秩决定；数值计算中通常避免显式求逆。

## 向量空间与特征结构

子空间需对加法和数乘封闭，基是一组线性无关且能张成空间的向量，维数是基中向量个数。列空间、零空间和秩刻画线性映射的能力。特征值和特征向量描述变换保持方向的分量，对角化可简化幂和动态系统分析。

## 正交、分解与应用

内积定义长度和夹角，正交投影给出最小二乘近似。QR 分解适合求解最小二乘，奇异值分解适用于任意矩阵，可用于降维、压缩、去噪和条件分析。软件计算应关注浮点误差、条件数、稀疏性和算法稳定性，而不只关注代数公式。

# 计算机组成原理

## 数据表示与指令系统

计算机用二进制表示整数、浮点数、字符和指令。补码统一有符号整数加减，浮点数由符号、指数和尾数组成并存在舍入误差。指令集规定操作、寄存器、寻址方式和编码，是软件与硬件的接口；汇编能帮助理解函数调用、栈帧和数据布局。

## 处理器与存储层次

处理器数据通路包含寄存器、运算部件和互连，控制器决定每条指令的数据流。流水线提高吞吐率，但会遇到结构、数据和控制相关，需要转发、停顿和分支预测。寄存器、缓存、主存和外存构成存储层次，局部性让小而快的缓存能够降低平均访问时间。

## 输入输出与性能

总线和控制器连接处理器、存储器与外设，中断让设备异步请求服务，DMA 减少大块数据传输时的处理器参与。虚拟存储把地址转换与保护结合起来。性能比较应结合执行时间、吞吐量、CPI、时钟频率、缓存未命中和并行度，避免只用单一硬件参数判断。

# 操作系统

## 进程、线程与调度

进程是资源与隔离的容器，线程是处理器调度的执行流。上下文切换保存并恢复执行状态，调度算法在响应时间、吞吐量、公平性和实时性之间权衡。系统调用让用户程序受控地请求内核服务，用户态与内核态提供权限隔离。

## 同步、通信与死锁

并发访问共享状态会产生竞态，互斥锁、信号量、条件变量和消息传递用于协调。正确同步需要定义临界区、状态条件和内存可见性。死锁的四个必要条件是互斥、占有并等待、不可抢占和循环等待，可通过预防、避免、检测或恢复处理。

## 内存、文件与保护

分页和分段管理地址空间，虚拟内存通过页表与页面置换扩大可用空间并隔离进程。文件系统组织文件、目录、元数据和空闲空间，缓存与日志改善性能和崩溃一致性。设备驱动抽象硬件差异，权限、访问控制和虚拟化构成系统保护与资源复用基础。

# Linux系统与应用

## 文件、权限与 Shell

Linux 将大量资源呈现为文件，通过目录形成统一命名空间。相对路径、绝对路径、链接、挂载和文件描述符是基本概念。读写执行权限分别作用于所有者、组和其他用户，sudo 应按最小权限使用。Shell 负责命令解析、变量、重定向、管道与进程启动。

## 进程、服务与网络

进程具有 PID、父子关系、环境和资源限制，可用 ps、top、kill 等工具观察与控制。systemd 管理服务生命周期、依赖和日志。包管理器负责软件安装与更新；ip、ss、curl、dig 和防火墙工具帮助检查接口、路由、端口、名称解析与连接问题。

## 脚本、运维与安全

Shell 脚本适合自动化命令编排，应正确引用变量、检查退出码并安全处理输入。grep、sed、awk、find 和管道可组合文本处理流程。运维需要监控 CPU、内存、磁盘、网络和日志，建立备份与恢复演练；及时更新、限制暴露端口、保护 SSH 密钥并审计特权操作。

# 概率论与数理统计

## 概率与随机变量

样本空间和事件描述随机试验，条件概率刻画已有信息下的可能性，独立性表示一个事件不改变另一个事件概率。贝叶斯公式用于根据证据更新判断。随机变量把结果映射为数值，分布函数、概率质量或密度描述其规律，期望和方差概括中心与波动。

## 常见分布与极限定理

伯努利、二项、泊松、均匀、指数和正态分布适用于不同生成机制。协方差和相关系数描述线性共同变化，但相关不等于因果。大数定律解释样本平均趋近总体期望，中心极限定理说明许多独立贡献之和在适当条件下近似正态。

## 统计推断与软件应用

抽样分布连接样本与总体，点估计应考虑无偏性、一致性和效率，置信区间表达估计不确定性。假设检验需事先定义原假设、显著性水平和统计量，并同时关注效应大小、检验功效与多重比较。回归可建模变量关系，但必须检查残差、异常值、共线性和适用假设。

# 计算机网络

## 分层与链路通信

分层把通信任务拆成可替换的协议，TCP/IP 模型通常包含链路、网际、传输和应用层。以太网帧、MAC 地址、交换与 ARP 支持局域网通信；无线网络还要处理共享介质、干扰和移动性。封装让每层添加本层控制信息。

## IP、路由与传输

IP 提供尽力而为的数据报传输，子网掩码和前缀确定网络范围，路由器依据路由表转发。ICMP 用于差错与诊断，NAT 转换地址和端口。UDP 提供轻量无连接数据报；TCP 通过序号、确认、重传、流量控制和拥塞控制提供可靠字节流。

## 应用协议与网络安全

DNS 解析名称，HTTP/HTTPS 传输 Web 资源，TLS 提供身份认证、机密性和完整性。电子邮件、SSH、DHCP 等协议服务于不同应用。排障应从物理连接、地址配置、路由、端口、名称解析到应用逐层定位，并用抓包验证假设。网络安全还需防范窃听、伪造、拒绝服务和错误配置。

# 编译原理

## 词法与语法分析

编译器把源程序转换为目标程序，并在各阶段维护符号与诊断信息。词法分析用正则语言识别关键字、标识符和字面量，有限自动机是其理论基础。语法分析依据上下文无关文法构建语法树，常见方法包括递归下降、LL、LR 及其变体。

## 语义与中间表示

语义分析检查声明、作用域、类型、调用和控制流约束，符号表记录名字的属性。抽象语法树去除无关语法细节，中间表示将前端语言与后端机器解耦。三地址码、基本块、控制流图和 SSA 形式便于数据流分析与优化。

## 优化、代码生成与运行时

优化必须保持程序可观察语义，可执行常量传播、死代码消除、公共子表达式消除、循环优化和内联。代码生成负责指令选择、寄存器分配、调度和目标格式。运行时系统处理调用约定、栈、堆、异常和垃圾回收。实现时应通过测试语料、差分测试和清晰错误定位验证各阶段。

# 移动应用开发

## 平台、生命周期与界面

移动应用运行在受资源和生命周期约束的平台上。Android 的 Activity、Service 等组件与 iOS 的应用和视图生命周期都会因切换、旋转或系统回收而变化。界面应适配不同尺寸、密度、方向和无障碍需求，状态需要区分临时 UI 状态与可持久业务状态。

## 数据、网络与架构

本地存储可采用键值、文件或数据库，敏感数据应使用平台安全存储。网络请求需要处理超时、重试、缓存、离线、分页和弱网，并避免阻塞主线程。分层架构、单向数据流和依赖注入有助于隔离界面、领域逻辑与数据源；跨平台方案仍需理解平台能力差异。

## 权限、测试与发布

权限应在真正需要时申请并解释用途，最小化收集和后台活动。深链、通知、WebView 与第三方 SDK 都可能扩大攻击面。测试应覆盖业务单元、组件交互、界面流程、设备兼容与性能。发布前还需处理签名、版本、隐私声明、崩溃监控、灰度分发和应用商店规则。

# 游戏软件开发

## 游戏循环与数学基础

游戏循环持续处理输入、更新世界并渲染画面。固定时间步适合物理模拟，可变时间步需使用 delta time 保持不同帧率下速度一致。向量、矩阵、四元数和坐标变换支持位置、方向、相机与骨骼动画，空间划分用于降低碰撞和可见性计算成本。

## 引擎系统与玩法

渲染管线负责几何、材质、光照和后处理，物理系统处理碰撞检测、约束和响应。实体组件系统通过组合数据与行为管理大量对象。动画状态机、寻路、行为树和有限状态机常用于角色表现与 AI。玩法代码应将规则、表现和输入解耦，便于测试和平衡调整。

## 网络、性能与生产流程

联网游戏需要处理权威状态、延迟、丢包、插值、预测、回滚和反作弊。性能优化应通过分析器定位 CPU、GPU、内存、加载或网络瓶颈，再采用对象池、批处理、细节层次和异步加载。版本控制需适配大型二进制资源，编辑器工具、自动构建、资源校验、可玩性测试和遥测共同支撑迭代生产。
