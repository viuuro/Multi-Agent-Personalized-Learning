## 第1章 计算机系统概论

### 1.1 冯·诺依曼体系结构

计算机由 **五大部件** 组成：输入设备、输出设备、存储器、运算器、控制器。核心思想是 **存储程序**——指令和数据以同等地位存放于存储器中，按地址寻访，均以二进制表示。

| 特点 | 说明 |
|------|------|
| 五大部件 | 输入/输出/存储/运算/控制 |
| 存储程序 | 程序和数据在同一存储器，二进制表示 |
| 指令格式 | 操作码 + 地址码 |
| 顺序执行 | 由 PC（程序计数器）控制，可条件跳转 |
| 以运算器为中心 | 早期设计；**现代以存储器为中心** |

### 1.2 哈佛架构 vs 冯·诺依曼架构

| 维度 | 哈佛架构 | 冯·诺依曼架构 |
|------|---------|--------------|
| 指令/数据存储 | **分开存储** | **同一存储** |
| 带宽 | 更高（可并行取指+取数） | 可能产生瓶颈 |
| 应用 | DSP、微控制器 | 通用计算机 |

### 1.3 计算机系统的层次结构

```
高级语言 → 汇编语言 → 操作系统 → 机器指令 → 微指令 → 逻辑电路
```

- **ISA（指令集架构）**：软件可见的规范——指令、寄存器、数据类型、内存模型
- **微体系结构（组织）**：具体硬件实现——Cache大小、流水线级数、执行单元数量
- 相同 ISA 可以有不同微体系结构实现

### 1.4 关键寄存器速查

| 缩写 | 全称 | 功能 | 程序员可见？ |
|------|------|------|:---:|
| PC | 程序计数器 | 存放下条指令地址 | ✓ |
| IR | 指令寄存器 | 存放当前指令 | ✗ |
| MAR | 存储器地址寄存器 | 寻址 | ✗ |
| MDR | 存储器数据寄存器 | 暂存数据 | ✗ |
| ACC | 累加器 | 存放操作数/结果 | ✓ |
| ALU | 算术逻辑单元 | 运算核心 | ✓ |
| CU | 控制单元 | 发控制信号 | ✗ |
| PSW/FLAGS | 程序状态字 | 条件标志（OF/SF/ZF/CF） | ✓ |

### 1.5 性能度量

| 指标 | 公式 | 说明 |
|------|------|------|
| CPU执行时间 | T = IC × CPI × T_cycle | IC=指令数，CPI=每指令周期数 |
| CPI | CPI = 总时钟周期数 / 指令数 | 与系统结构有关，与频率无关 |
| MIPS | MIPS = f / (CPI × 10⁶) | 每秒百万条指令 |
| MFLOPS | MFLOPS = 浮点操作次数 / (T × 10⁶) | 浮点性能专用 |
| Amdahl定律 | S = 1 / ((1-p) + p/s) | p=可加速比例, s=加速倍数 |

### 1.6 八大设计思想

1. **性能 via 流水线** — 提升吞吐量
2. **可靠性 via 冗余** — 容错设计
3. **性能 via 预测** — 分支预测提前执行
4. **加速常见情况** — 优化最频繁路径（Amdahl定律的推论）
5. **存储器层次结构** — 缓存→内存→外存金字塔
6. **性能 via 并行** — 指令级/数据级/线程级并行
7. **遵循摩尔定律设计** — 预判晶体管增长
8. **使用抽象简化设计** — 分层降复杂度

## 第2章 数据的表示和运算

### 2.1 进位计数制

- **二进制(B)**、**八进制(O)**、**十进制(D)**、**十六进制(H)**
- BCD码：8421码（有权）、余三码（无权=8421+3）、2421码（≥5时最高位为1）
- ASCII：7位编码；UTF-8：变长1-4字节

### 2.2 五种码制对比

| 属性 | 原码 | 补码 | 反码 | 移码 |
|------|------|------|------|------|
| 符号位 | 0正1负 | 0正1负 | 0正1负 | **1正0负** |
| +0 | 0000 | 0000 | 0000 | 1000 |
| -0 | 1000 | 无（= -2ⁿ） | 1111 | 0000（= -2ⁿ） |
| 8位范围 | -127~127 | **-128~127** | -127~127 | -128~127 |
| 用途 | 浮点尾数 | **整数运算** | 过渡码制 | **浮点阶码** |
| 加法 | 需判断符号 | 直接加 | 循环进位 | — |

### 2.3 溢出判断三种方法

| 方法 | 条件 |
|------|------|
| 法一 | 同号相加，结果符号与加数不同 → 溢出 |
| 法二 | 数值最高位进位 Cₙ ≠ 符号位进位 C_f → 溢出 |
| 法三 | 双符号位（变形补码）fₛ₁ ≠ fₛ₂ → 溢出 |

### 2.4 定点数与浮点数

| 类型 | 32位范围 |
|------|---------|
| 定点整数 | -2³¹ ~ 2³¹-1 |
| 定点小数 | -1 ~ 1-2⁻³¹ |
| 单精度浮点 | -2¹²⁷ ~ (1-2⁻²³)×2¹²⁷ |
| 双精度浮点 | -2¹⁰²³ ~ (1-2⁻⁵²)×2¹⁰²³ |

### 2.5 IEEE 754 浮点数格式

| 精度 | 总位数 | 符号 | 阶码 | 尾数 | 偏移量 |
|------|:---:|:---:|:---:|:---:|:---:|
| 单精度 | 32 | 1 | 8 | 23 | 127 |
| 双精度 | 64 | 1 | 11 | 52 | 1023 |

**特殊值**：
- 阶码全0 + 尾数全0 → **零**
- 阶码全1 + 尾数全0 → **无穷大**
- 阶码全1 + 尾数非0 → **NaN**
- 阶码全0 + 尾数非0 → **非规格化数（Denormal）**

### 2.6 浮点运算五步流程

1. **对阶** — 小阶向大阶对齐（ΔE补偿尾数）
2. **尾数加减**
3. **规格化** — 符号位 ≠ 数值最高位
4. **舍入** — 四种模式：就近/向零/向正无穷/向负无穷
5. **判溢出** — 检查阶码

### 2.7 算术运算单元

| 运算器 | 核心思想 | 时间复杂度 |
|--------|---------|:---:|
| 行波进位加法器(RCA) | 逐级进位 | O(n) |
| 进位选择加法器(CSA) | 高低位并行计算两种结果再选择 | O(√n) |
| 超前进位加法器(CLA) | 并行计算进位 | O(log n) |
| 迭代乘法器 | 逐位移位加 | O(n) |
| Booth乘法器 | 减少部分积 | O(n) |
| Wallace树乘法器 | 并行压缩 | O(log n) |
| 恢复余数除法 | 尝试减→恢复 | O(n) |
| 不恢复余数除法 | 加减交替 | O(n) |
| Goldschmidt除法 | 级数收敛 | O(log n) |
| Newton-Raphson除法 | 迭代逼近 | O(log n) |

### 2.8 校验码

| 校验码 | 码距 | 能力 | 关键公式 |
|--------|:---:|------|-------------|
| 奇偶校验 | 2 | 查1位/奇位错 | 1位校验位 |
| 海明码 | 4 | 纠1位错，查2位错 | 2^r ≥ k+r+1 |
| CRC | 可变 | 发现/纠正连续多位错 | 模2运算 |

### 2.9 C语言数据类型与转换

| 类型 | 32位大小 | 64位大小 |
|------|:---:|:---:|
| char | 1 | 1 |
| short | 2 | 2 |
| int | 4 | 4 |
| long | 4 | **8** |
| float | 4 | 4 |
| double | 8 | 8 |
| 指针 | 4 | **8** |

- **大端**：高字节在低地址；**小端**：低字节在低地址
- 强制类型转换：短→长扩展符号位；长→短截断；int↔float 需经过中间格式

## 第3章 存储系统

### 3.1 存储器层次结构

```
寄存器(L1) → Cache(L2/L3) → 主存(DRAM) → 辅存(SSD/HDD)
速度递减 ←—————————————— 容量递增 ——————————————→
```

- **时间局部性(Temporal Locality)**：刚访问的数据可能再次访问
- **空间局部性(Spatial Locality)**：刚访问数据附近的数据可能被访问
- 层次结构的基本原理：**加速常见情况 + 局部性原理**

### 3.2 SRAM vs DRAM

| 维度 | SRAM | DRAM |
|------|------|------|
| 存储单元 | 6管触发器 | 1管+1电容 |
| 速度 | 快 | 慢（~5-10x） |
| 密度 | 低 | 高 |
| 功耗 | 低（静态） | 需刷新 |
| 刷新 | 不需要 | **≤2ms 必须刷新** |
| 用途 | Cache | 主存 |
| 成本 | 高 | 低 |

**DRAM刷新三种方式**：集中刷新、分散刷新、异步刷新

### 3.3 Cache 地址映射

| 方式 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **直接映射** | 每个主存块→固定Cache行 | 简单、快 | 冲突多、利用率低 |
| **全相联映射** | 任意主存块→任意Cache行 | 灵活、冲突少 | 比较器成本高 |
| **组相联映射** | 先分组，组内全相联 | **折衷方案** | — |

### 3.4 Cache 写策略

| 策略 | 写入行为 | 优点 | 缺点 |
|------|---------|------|------|
| **写通法(Write-Through)** | 同时写Cache和主存 | 简单、一致性好 | 写开销大 |
| **写回法(Write-Back)** | 仅写Cache，替换时写回 | 写开销小 | 需脏位管理 |

- **写分配(Write-Allocate)**：写缺失时先调入再写
- **非写分配(No-Write-Allocate)**：写缺失时直接写主存

### 3.5 Cache 替换算法

| 算法 | 原理 |
|------|------|
| **LRU** | 替换最久未使用的块 |
| **FIFO** | 替换最早进入的块 |
| **随机(Random)** | 随机选择 |
| **LFU** | 替换使用次数最少的块 |
| **伪LRU** | 近似LRU，实现简单 |

### 3.6 平均访存时间 (AMAT)

```
AMAT = t_hit + miss_rate × miss_penalty
```

### 3.7 虚拟存储器

| 概念 | 说明 |
|------|------|
| **页(Page)** | 虚拟空间固定大小块（通常4KB） |
| **页框(Frame)** | 物理空间对应块 |
| **页表(Page Table)** | 虚→实地址映射表 |
| **TLB** | 页表快表，加速地址转换 |
| **缺页(Page Fault)** | 访问不在主存的页，触发OS处理 |
| **MMU** | 内存管理单元，硬件地址转换 |

### 3.8 多体交叉存储器

- **低位交叉**：连续地址分布在不同模块，支持流水访问
- **高位交叉**：连续地址在同一模块
- 第i模块地址 = M × j + i（M=模块数）

### 3.9 其他存储器

| 类型 | 特点 |
|------|------|
| **ROM** | 出厂掩模，不可改写 |
| **PROM** | 用户一次编程 |
| **EPROM** | 紫外擦除，可多次编程 |
| **EEPROM** | 电擦除 |
| **Flash** | 块擦除，NAND/NOR |
| **CAM(内容可寻址)** | 按内容查找，用于TLB/Cache Tag |
| **SSD** | 基于NAND Flash，比HDD快但有限写入次数 |

## 第4章 指令系统

### 4.1 指令格式

指令 = **操作码(Opcode)** + **地址码(Operand)**

- **操作码扩展**：高频指令短操作码，低频指令长操作码
- **定长指令**：所有指令等长（RISC特征）
- **变长指令**：指令长度可变（CISC特征，如x86 1~15字节）

### 4.2 寻址方式（8种）

| # | 寻址方式 | 含义 | 访存次数 |
|:---:|------|------|:---:|
| 1 | 隐含寻址 | 操作数在指定寄存器 | 0 |
| 2 | 立即寻址 | 操作数在指令中 | 0 |
| 3 | 直接寻址 | EA = 指令中地址 | 1 |
| 4 | 间接寻址 | EA = [指令中地址] | ≥2 |
| 5 | 寄存器寻址 | 操作数在寄存器 | 0 |
| 6 | 寄存器间接寻址 | EA = (寄存器) | 1 |
| 7 | 变址寻址 | EA = (变址寄存器) + 偏移量 | 1 |
| 8 | 基址寻址 | EA = (基址寄存器) + 偏移量 | 1 |
| 9 | 相对寻址 | EA = (PC) + 偏移量 | 1 |
| 10 | 堆栈寻址 | 栈顶操作 | 0~1 |

> **基址 vs 变址区别**：基址面向系统（程序重定位），变址面向用户（数组访问）

### 4.3 CISC vs RISC（九维度对比）

| 维度 | CISC | RISC |
|------|------|------|
| 指令数量 | 多(200+) | 少(<100) |
| 指令长度 | 变长 | 定长 |
| 指令格式 | 多种 | 少(3~4种) |
| 寻址方式 | 多(>10) | 少(≤5) |
| 访存指令 | 任意指令可访存 | **仅Load/Store** |
| 执行方式 | 微程序控制 | **硬布线控制**(快) |
| 流水线 | 困难 | 容易 |
| 编译优化 | 简单 | 复杂 |
| 代表 | x86, VAX | ARM, RISC-V, MIPS |

### 4.4 RISC-V 指令格式

| 格式 | 用途 | 字段 |
|------|------|------|
| **R-type** | 寄存器-寄存器运算 | funct7+rs2+rs1+funct3+rd+opcode |
| **I-type** | 立即数运算/Load | imm[11:0]+rs1+funct3+rd+opcode |
| **S-type** | Store | imm[11:5]+rs2+rs1+funct3+imm[4:0]+opcode |
| **B-type** | 条件分支 | imm[12|10:5]+rs2+rs1+funct3+imm[4:1|11]+opcode |
| **U-type** | LUI/AUIPC | imm[31:12]+rd+opcode |
| **J-type** | 跳转(JAL) | imm[20|10:1|11|19:12]+rd+opcode |

### 4.5 ARMv8(LEGv8) 指令集特点

- 31个通用寄存器(X0-X30) + SP + 零寄存器XZR
- Load-Store架构
- 支持条件执行(Predication)
- 支持寄存器移位操作内嵌在ALU指令中

### 4.6 x86 指令集特点

- 变长指令(1-15字节)
- 丰富的寻址模式（基址+变址×比例+偏移）
- 双操作数格式（目的操作数同时也是源之一）
- 支持SIMD扩展(SSE/AVX)

## 第5章 中央处理器(CPU)

### 5.1 CPU 基本结构

```
┌─────────────────────────────────┐
│            CPU                   │
│  ┌──────────┐  ┌──────────────┐ │
│  │ 控制器(CU) │  │ 运算器(ALU)   │ │
│  │ PC→IR→译码│  │ ACC+ALU+PSW │ │
│  └──────────┘  └──────────────┘ │
│  ┌──────────────────────────┐   │
│  │      寄存器堆(Register File) │   │
│  └──────────────────────────┘   │
└─────────────────────────────────┘
```

### 5.2 三级时序

| 周期 | 定义 |
|------|------|
| **时钟周期**(T周期) | 基准脉冲，CPU最小时间单位 |
| **机器周期**(CPU周期) | 完成一个基本操作(如取指)的时间 |
| **指令周期** | 取指+译码+执行+写回的完整时间 |

### 5.3 指令执行五阶段

| 阶段 | 缩写 | 操作 |
|------|------|------|
| **取指** | IF | PC→MAR→M→MDR→IR, PC+4 |
| **译码** | ID | IR译码，读寄存器 |
| **执行** | EX | ALU运算，地址计算 |
| **访存** | MEM | Load/Store访问数据存储器 |
| **写回** | WB | 结果写回寄存器 |

### 5.4 微程序控制 vs 硬布线控制

| 维度 | 微程序控制 | 硬布线控制 |
|------|-----------|-----------|
| 实现方式 | 控存ROM中的微指令序列 | 组合逻辑电路 |
| 速度 | 慢 | **快** |
| 灵活性 | 可改（动态微程序设计） | 固定 |
| 规整性 | 规整 | 复杂 |
| 应用 | CISC | **RISC** |
| 微指令格式 | 水平型(长) vs 垂直型(短) | — |

### 5.5 流水线(Pipeline)

#### 基本概念
- **吞吐率**：单位时间完成指令数
- **加速比**：非流水时间 / 流水时间
- **效率**：流水线利用率

#### 三大冒险(Hazards)

| 类型 | 原因 | 解决方法 |
|------|------|---------|
| **结构冒险** | 硬件资源冲突 | 增加资源/分离I-Cache与D-Cache |
| **数据冒险** | 指令间数据依赖(RAW/WAW/WAR) | **转发**(Forwarding) + **阻塞**(Stalling) |
| **控制冒险** | 分支/跳转 | 分支预测 + 延迟槽 |

#### 数据冒险详解

| 类型 | 含义 | 说明 |
|------|------|------|
| RAW(读后写) | 后指令读前，前指令未写完 | **最常见**，必须处理 |
| WAR(写后读) | 后指令写前，前指令未读完 | 乱序执行中需处理 |
| WAW(写后写) | 后指令与前指令写同目的地 | 乱序执行中需处理 |

#### 转发(Forwarding)路径
- EX→EX：执行阶段直接转发
- MEM→EX：访存阶段转发到执行阶段
- 转发不能解决所有RAW（如Load后立即使用的Load-Use Hazard需插入气泡）

#### 分支预测
- **静态预测**：总是预测不跳转 / 总是预测跳转 / BTFN(Backward Taken, Forward Not Taken)
- **动态预测**：
  - 1-bit预测器 → 2-bit饱和计数器 → 两级自适应预测器(Gshare/Tournament)
  - **BTB**(Branch Target Buffer)：缓存分支目标地址
  - **RAS**(Return Address Stack)：预测函数返回地址

#### 高级流水线技术

| 技术 | 说明 |
|------|------|
| **超标量(Superscalar)** | 每周期发射多条指令 |
| **乱序执行(OoO)** | 指令按数据就绪而非程序序执行 |
| **VLIW/EPIC** | 编译器静态调度，显式并行 |
| **同时多线程(SMT)** | 同一核心多线程共享执行单元 |

### 5.6 中断与异常

| 维度 | 中断(Interrupt) | 异常(Exception) |
|------|:---:|:---:|
| 来源 | **外部**（I/O设备、时钟） | **内部**（指令执行） |
| 同步性 | 异步 | 同步 |
| 可屏蔽性 | 可屏蔽/不可屏蔽(INTR/NMI) | 不可屏蔽 |
| 返回行为 | 下一条指令 | 当前指令或下一条 |
| 典型例子 | 键盘输入、DMA完成 | 除零、缺页、非法指令、断点 |

**精确异常(Precise Exception)**：异常发生时，之前的指令全部完成，之后的指令全部未执行。是乱序处理器设计的关键挑战。

**中断处理流程**：
```
保护现场(压栈) → 中断服务 → 恢复现场(弹栈) → 中断返回(RETI/IRET)
```

## 第6章 总线

### 6.1 总线分类

| 分类 | 传输内容 | 方向 |
|------|---------|:---:|
| 数据总线 | 数据 | 双向 |
| 地址总线 | 地址 | 单向(CPU→设备) |
| 控制总线 | 控制/状态信号 | 混合 |

### 6.2 总线性能

| 指标 | 公式 |
|------|------|
| 总线带宽 | 总线宽度 × 总线频率 |
| 实际传输率 | 总线带宽 × 有效利用率 |

### 6.3 定时方式

| 方式 | 特点 | 优点 | 缺点 |
|------|------|------|------|
| **同步定时** | 统一时钟控制 | 简单 | 设备速度必须匹配 |
| **异步定时** | 握手信号(Req/Ack) | 兼容不同速度设备 | 复杂 |
| **半同步定时** | 同步+等待信号 | 折衷 | — |
| **源同步** | 时钟随数据发送 | 高速 | — |

### 6.4 仲裁方式

| 方式 | 说明 |
|------|------|
| 集中式 | 菊花链、轮询、独立请求 |
| 分布式 | 各设备自行仲裁 |

### 6.5 总线标准

| 标准 | 类型 | 特点 |
|------|------|------|
| **PCI Express(PCIe)** | 串行点对点 | 分层协议、多通道(Lane)、高带宽 |
| **SATA** | 串行 | 硬盘接口 |
| **USB** | 串行 | 即插即用、热插拔、供电 |
| **FireWire(IEEE 1394)** | 串行 | 等时传输 |

### 6.6 信号编码

| 编码 | 原理 | 特点 |
|------|------|------|
| NRZ | 1=高,0=低 | 简单但无时钟 |
| NRZI | 跳变=1,不变=0 | USB使用 |

| RZ | 后半归零 | 自带时钟 |
| LVDS | 差分信号 | 高速、低功耗、抗干扰 |

## 第7章 输入输出系统

### 7.1 I/O 接口功能

1. 地址译码和设备选择
2. 数据缓冲和锁存
3. 信号电平转换
4. 传送控制与状态反馈
5. 中断管理

### 7.2 三种 I/O 控制方式对比

| 维度 | 程序查询 | 程序中断 | DMA |
|------|---------|---------|-----|
| 数据传送单位 | 字/字节 | 字/字节 | **块/批量** |
| CPU参与度 | 全程 | 传送时 | **仅开始和结束** |
| CPU效率 | 低 | 中 | **高** |
| 实现复杂度 | 最简单 | 中等 | 复杂 |
| 适用场景 | 低速设备 | 中低速 | **高速设备** |
| 额外硬件 | 无 | 中断控制器 | **DMAC** |
| 对程序透明性 | 不透明 | 半透明 | **透明** |

### 7.3 中断

- **中断向量**：每个中断源有唯一的向量号→对应服务程序入口
- **中断优先级**：高优先级的可打断低优先级（**中断嵌套**）
- **中断屏蔽**：通过中断允许触发器控制
- **中断请求/响应**：INTR(可屏蔽) vs NMI(不可屏蔽)

### 7.4 DMA

- **DMAC(DMA控制器)**：接管总线控制权
- **三种工作方式**：
  - CPU暂停方式：DMA期间CPU停止
  - 周期窃取(Cycle Stealing)：交替使用总线
  - 交替访问：CPU与DMA分时
- **传送过程**：预处理 → 数据传送 → 后处理

### 7.5 I/O 编址方式

| 方式 | 说明 | 代表 |
|------|------|------|
| **独立编址(I/O映射)** | 专用I/O指令，独立地址空间 | x86(in/out) |
| **统一编址(内存映射)** | I/O与内存共用地址空间 | ARM, RISC-V |

| # | 误区 | 正解 |
|:---:|------|------|
| 1 | "64位CPU"="所有数据固定64位" | 64位指通用寄存器/地址宽度，数据仍有8/16/32/64位 |
| 2 | 主频高必然更快 | 同样架构才可比，不同CPI/流水线效率影响更大 |
| 3 | 补码的-0=1000 | 补码中0**唯一表示**，10000000表示的是-128(8位) |
| 4 | Cache大必然好 | 过大会增加命中时间，可能降低性能 |
| 5 | RISC总是快于CISC | 现代x86(CISC)内部翻译为类RISC微操作，性能相当 |
| 6 | "流水线级数越多越好" | 级数增加→冒险代价增大→效率可能降低 |
| 7 | 浮点能精确表示所有实数 | 浮点表示有限精度，存在舍入误差 |
| 8 | 中断和异常是同一概念 | 中断=外部异步，异常=内部同步 |
| 9 | DMA不需要CPU参与 | DMA仍需CPU做**预处理和后处理** |
| 10 | 大端/小端只影响字节 | 也影响位的编号方式 |
