## 第1章 绪论与复杂度分析

### 1.1 数据结构定义

**数据结构**是相互之间存在一种或多种特定关系的数据元素的集合。包含三要素：逻辑结构、存储结构（物理结构）、数据运算。

逻辑结构分为四类：**集合**（元素同属一个集合）、**线性结构**（一对一）、**树形结构**（一对多）、**图状/网状结构**（多对多）。

存储结构分为四种：**顺序存储**（连续内存，如数组）、**链式存储**（指针连接，如链表）、**索引存储**（附加索引表）、**散列存储**（通过散列函数计算地址）。

### 1.2 算法的五个特征

有穷性、确定性、可行性、输入（零个或多个）、输出（至少一个）。

好算法的标准：正确性、可读性、健壮性、高效率与低存储。

### 1.3 时间复杂度

**大O记号**：上界（最坏保证）。**Omega**：下界。**Theta**：紧确界。

增长率排序（慢到快）：O(1) < O(log n) < O(sqrt n) < O(n) < O(n log n) < O(n^2) < O(n^3) < O(2^n) < O(n!)

三种时间情形：
- **最坏**：对任何输入都成立的上界。快排退化 O(n^2)。
- **平均**：需要概率分布假设。快排随机枢轴 O(n log n)。
- **摊还**：一串操作的总代价平均。动态数组扩容。摊还不等于平均，摊还是对任意操作序列的保证。

摊还分析三法：聚合分析（总代价除以次数）、记账法（便宜操作预存信用）、势能法（衡量结构势能变化）。

### 1.4 递推式与主定理

T(n) = aT(n/b) + f(n)。比较 f(n) 与 n^(log_b a)：
- f(n) 更小：T(n) = Theta(n^(log_b a))
- 同级：T(n) = Theta(n^(log_b a) * log n)
- f(n) 更大 + 正规条件：T(n) = Theta(f(n))

经典应用：归并排序 T(n)=2T(n/2)+n -> Theta(n log n)

### 1.5 常见误区

- O(2n) = O(n)，忽略常数因子
- 小n下常数因子重要，低复杂度不一定更快
- 空间复杂度只计辅助空间，不计输入输出本身
- 递归深度消耗调用栈，不计入O(1)空间

## 第2章 线性表

### 2.1 顺序表

用连续内存存储。随机访问 O(1)。插入删除需移动元素，平均移动 n/2 个。

动态数组扩容：倍增策略（x2）摊还 O(1)；增量策略（+固定大小）O(n)。

### 2.2 链表

单链表：节点含数据+next指针。头插法建表得到逆序序列。原地反转用三指针法（prev, curr, next）。

双链表：prev+next，删除已知节点 O(1)。

循环链表：判空不是 p.next==NULL，而是 p.next==head。

静态链表：用游标代替指针。

哨兵节点：统一头尾边界处理。

### 2.3 链表操作黄金法则

先保存后修改。区分前提（已知节点/已知前驱/已知下标，复杂度完全不同）。检查边界（头/尾/空）。

快慢指针应用：检测环（快2慢1相遇）、找中点（快到达尾部时慢在中点）。

### 2.4 顺序表 vs 链表

| 需求 | 顺序表 | 链表 |
|------|:--:|:--:|
| 频繁随机访问 | Yes | No |
| 频繁头部插入删除 | No | Yes |
| 缓存友好 | Yes | No |
| 动态增长 | No | Yes |

## 第3章 栈、队列与数组

### 3.1 栈

LIFO。自然对应函数调用返回、括号匹配、撤销操作。

**卡特兰数**：n个元素入栈，出栈排列数 C_n = C(2n,n)/(n+1)。C_3=5, C_4=14, C_5=42。

**单调栈**：栈内元素始终单调，每元素至多进出一次 O(n)。应用：找下一个更大元素。

**共享栈**：两栈共享一个数组空间，判满条件 top1+1==top2。

### 3.2 队列

FIFO。循环队列的 front/rear 含义必须统一。判空/满方案：
- 牺牲一个单元：(rear+1)%M==front 判满，front==rear 判空
- size字段：size==0判空，size==M判满
- tag标记：区分 front==rear 是空还是满

双端队列：允许两端插入和删除。

### 3.3 特殊矩阵的压缩存储

- **对称矩阵**：只存下三角或上三角，n(n+1)/2个元素
- **三角矩阵**：上/下三角+常数，n(n+1)/2+1个元素
- **对角矩阵**（三对角）：3n-2个非零元素
- **稀疏矩阵**：三元组或十字链表存储

## 第4章 串

### 4.1 基本概念

串是由零个或多个字符组成的有限序列。空串长度为0，空格串长度>=1。子串是串中任意连续字符组成的子序列。

### 4.2 模式匹配算法

**BF算法**：逐个比较，失配时主串回溯。最坏 O(n*m)。

**KMP算法**：主串不回溯，利用next数组。复杂度 O(n+m)。next[j] = T[0..j-1]的最长相等真前后缀长度。next[0] = -1。

示例："ababaca" -> next = [-1, 0, 0, 1, 2, 3, 0]

**Boyer-Moore算法**（Drozdek 13.1.3）：从右向左匹配。坏字符规则（失配字符上次出现位置决定右移）+ 好后缀规则（已匹配后缀上次出现决定右移）。最好 O(n/m)。

**Rabin-Karp算法**：滚动哈希，适合多模式匹配。期望 O(n+m)。

**编辑距离**（Levenshtein距离，Drozdek 13.2）：通过插入/删除/替换将A变为B的最少操作数。DP解法 dp[i][j] 递推。

## 第5章 树与二叉树

### 5.1 树的基本术语

- 根、叶子、父/子/兄弟/堂兄弟
- 深度：自顶向下，根深为1（或0，需看教材定义）
- 高度：自底向上，叶子高为1（或0）
- 度：节点的子节点数；树的度 = max{节点度}
- 路径长度：边的数量

### 5.2 二叉树必记公式

| 性质 | 公式 |
|------|------|
| 叶子-度2关系 | **n0 = n2 + 1** |
| 第i层最多节点 | 2^(i-1) |
| 深度为k最多节点 | 2^k - 1 |
| n节点完全二叉树深度 | floor(log2 n) + 1 |
| n节点二叉链表空指针 | **n + 1** |
| 满二叉树定理（Shaffer 5.1.1） | 叶子数 = 内部节点数 + 1 |
| 线索二叉树线索数 | **n + 1** |

### 5.3 四种遍历

| 遍历 | 顺序 | 主要用途 |
|------|------|---------|
| 先序 | 根-左-右 | 序列化/复制树 |
| 中序 | 左-根-右 | BST得有序序列 |
| 后序 | 左-右-根 | 删除整棵树 |
| 层序 | 逐层 | BFS、无权最短路径 |

非递归中序遍历用栈辅助。Drozdek 6.4.3 有无栈深度优先（利用parent指针回溯）。

### 5.4 遍历与重建

仅有先序+后序**不能唯一确定**二叉树。先序+中序或后序+中序可确定（所有键不同时）。

重建方法：先序第一个是根 -> 中序中根左边左子树、右边右子树 -> 递归。

### 5.5 线索二叉树

n节点二叉链表有n+1空指针，利用为前驱/后继线索。需ltag/rtag区分指向孩子还是线索。

### 5.6 哈夫曼树

带权路径长度WPL最小的二叉树。构造：每次取两个最小权值合并（最小堆 O(n log n)）。总节点数 = 2*n0 - 1。哈夫曼编码是**最优前缀编码**。

### 5.7 树、森林与二叉树的转换

- 树 -> 二叉树：孩子兄弟表示法（左1stChild，右nextSibling）
- 树先根遍历 = 二叉树先序
- 树后根遍历 = **二叉树中序**（易错！）
- 森林先序 = 二叉树先序
- 森林中序 = 二叉树中序

### 5.8 并查集

支持 Union 和 Find 操作。路径压缩（Find时直指根）+ 按秩合并（小树入大树）-> m次操作摊还 O(m * alpha(n))，alpha(n)是反阿克曼函数，实践中 <= 4。

## 第6章 BST与平衡树

### 6.1 BST核心

不变量：**左子树所有值 < 根 < 右子树所有值**。中序遍历必严格递增。

查找/插入 O(h)，h为树高。最坏退化 O(n)（完全偏斜）。

### 6.2 BST删除

三种情况：
1. 叶子 -> 直接删
2. 单孩子 -> 孩子顶上
3. 双孩子 -> 中序后继（右子树最左）替换值，删除后继（至多一个右孩子）

注意：值被替换的节点不等于物理删除的节点。Drozdek 6.6 还有合并法和复制法两种实现。

### 6.3 AVL树

不变量：任意节点平衡因子（左高-右高）∈ {-1, 0, 1}。

| 旋转 | 口诀 |
|------|------|
| LL -> 右单旋 | 左孩子太重，向右扳 |
| RR -> 左单旋 | 右孩子太重，向左扳 |
| LR -> 先左后右 | 先转孙子，再转根 |
| RL -> 先右后左 | 先转孙子，再转根 |

高度 <= 1.44 log2(n+1)。查找密集型场景选用。

### 6.4 DSW算法（Drozdek 6.7.1）

不从零建树的平衡化：BST -> "葡萄藤"（仅右链）-> 旋转调整为平衡树。

### 6.5 红黑树

高度 <= 2 log2(n+1)。五性质：红黑二色、根必黑、叶(NIL)必黑、红不连续（红孩子必黑）、黑高一致。

| 对比 | AVL | 红黑树 |
|------|-----|--------|
| 平衡度 | <= 1.44log n | <= 2log n |
| 插入旋转 | <= 2次 | <= 2次 |
| 删除旋转 | 可O(log n)次 | <= 3次 |
| 应用 | 查找密集型 | 更新密集型（C++ std::map） |

### 6.6 Scapegoat树（Morin 第8章）

不旋转。子树"太不平衡"时拍平重建。查找 O(log n)，插入摊还 O(log n)。不需要节点存储平衡信息。删除用惰性标记。

### 6.7 伸展树（Drozdek 6.8）

被访问节点旋转移到根。自动适应用访问模式。摊还 O(log n)。无需显式平衡信息。

### 6.8 Treap（Drozdek 6.10）

每个节点有key（BST序）和priority（堆序）。随机优先级使树期望平衡。

### 6.9 B树家族

B树：数据在所有节点。B+树：数据仅叶节点，叶链串接（范围查询高效）。B*树：节点填充率 >= 2/3。2-3树：m=3的最简单B树变体。2-4树等价于红黑树。R-树：空间对象矩形索引（GIS）。

## 第7章 堆与优先队列

### 7.1 堆的性质

堆不是有序的（仅父<=子）。堆不是BST。是完全二叉树，数组存储无指针。小根堆的"最小"只在根。

### 7.2 堆操作

插入：放末尾 -> 上滤 O(log n)。删除：末元素替根 -> 下滤 O(log n)。

建堆 O(n)：从最后一个内部节点（n/2-1）向前依次下滤（Floyd法）。多数节点靠近叶层，下滤距离短。

堆排序 O(n log n)，不稳定。

## 第8章 散列表

### 8.1 核心组成

散列表 = 散列函数 + 冲突处理。好函数要快且分布均匀。装载因子 alpha = n/m。

### 8.2 冲突处理

**拉链法**：链表存冲突元素。alpha可>1。删除直接。ASL成功=1+alpha/2。缓存差。

**开放寻址**：线性探测（主聚集）、平方探测（需表长4k+3质数）、双散列。alpha必须<1。删除需墓碑标记。缓存好。

### 8.3 散列函数

除留余数法、乘法散列（Morin 5.1.1）、Tabulation散列（Morin 5.2.3，查表代算术，硬件友好）。

### 8.4 高级散列（Drozdek 10.4-10.6）

- **完美散列**：无冲突，适合静态键集
- **布谷鸟散列**：双函数，冲突踢走，可能死循环
- **可扩展散列**：目录+动态调整地址位
- **线性散列**：逐步分裂桶

### 8.5 散列码

基本类型直接映射。复合对象组合各字段。数组/字符串用多项式：s[0]*a^(n-1)+s[1]*a^(n-2)+...+s[n-1]（Morin 5.3.3）。

## 第9章 图

### 9.1 必记公式

| 公式 | 条件 |
|------|------|
| 无向完全图边数 | **n(n-1)/2** |
| 有向完全图边数 | **n(n-1)** |
| 连通图最少边 | n-1（无向），n（强连通有向） |
| 生成树边数 | n-1 |
| Sigma度 = 2|E| | 无向图 |
| DFS无横边 | 无向连通图 |

### 9.2 存储结构

**邻接矩阵**：O(V^2)空间，O(1)判边，适合稠密图。
**邻接表**：O(V+E)空间，O(degree)判边，适合稀疏图。
**十字链表**（有向图）、**邻接多重表**（无向图）。

### 9.3 遍历

| | BFS | DFS |
|--|-----|-----|
| 数据结构 | 队列 | 栈（递归/显式） |
| 邻接表复杂度 | O(V+E) | O(V+E) |
| 邻接矩阵复杂度 | O(V^2) | O(V^2) |

**Whatever-First Search统一框架**（Erickson 5.5）：DFS/BFS/Dijkstra本质同一算法，区别仅在bag结构——栈->DFS，队列->BFS，优先队列->BestFirst/Dijkstra。

### 9.4 最短路径

| 算法 | 范围 | 负权边 | 复杂度 |
|------|:--:|:--:|:--:|
| BFS | 无权单源 | — | O(V+E) |
| **Dijkstra** | 非负单源 | No | O((V+E)logV) |
| **Bellman-Ford** | 单源 | Yes | O(VE) |
| **Floyd** | 全源 | Yes（无负环） | O(V^3) |
| **Johnson** | 全源 | Yes | O(V^2logV+VE) |

> Dijkstra不能处理负权边：贪心假设"一旦距离确定就不会再更新"被负权打破。Bellman-Ford第V轮仍松弛说明有负环。

**The Only SSSP Algorithm**（Erickson 8.3）：所有单源最短路径=反复松弛。区别仅在松弛顺序。

**Johnson算法**（Erickson 9）：Bellman-Ford算势能 -> 重赋权消除负边 -> 每顶点Dijkstra。稀疏图优于Floyd。

### 9.5 最小生成树

| 算法 | 策略 | 复杂度 | 适合 |
|------|------|:--:|------|
| Prim | 一个分量向外扩展 | O(E log V) | 稠密 |
| Kruskal | 全局最小边逐一加入 | O(E log E) | 稀疏 |
| **Boruvka**（Erickson 7.3） | 所有分量同时选最小外出边 | O(E log V) | **并行** |

### 9.6 拓扑排序与关键路径

拓扑排序：Kahn（入度为0）和DFS逆完成序。仅DAG可排。

AOE网关键路径：ve(j)最早时间，vl(j)最迟时间。关键活动：e(i)==l(i)。

### 9.7 强连通分量（Erickson 6）

**Kosaraju-Sharir**（两次DFS）：原图DFS记录完成序 -> 反向图按完成序逆序DFS -> 每棵DFS树=SCC。O(V+E)。

**Tarjan**（一次DFS）：维护index和lowlink。lowlink[v]==index[v]时找到SCC根。O(V+E)。

### 9.8 网络流（Erickson 10-11）

**Ford-Fulkerson方法**：反复找增广路径。反向边=后悔机制。

**Edmonds-Karp**：最短增广路径 -> O(VE^2)。

**最大流五大应用**（Erickson 11）：二分图匹配、边不交路径、棒球淘汰分析、项目选择、考试排期。

### 9.9 欧拉图与哈密顿图

欧拉回路：每个顶点度为偶数（无向）-> P问题。哈密顿回路：NP-完全。

## 第10章 排序

### 10.1 九种算法总览

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|------|:--:|:--:|:--:|:--:|:--:|
| 直接插入 | O(n) | O(n^2) | O(n^2) | O(1) | Yes |
| 折半插入 | O(n log n) | O(n^2) | O(n^2) | O(1) | Yes |
| 希尔 | O(n) | ~O(n^1.3) | O(n^2) | O(1) | No |
| 冒泡 | O(n) | O(n^2) | O(n^2) | O(1) | Yes |
| 快速 | O(n log n) | O(n log n) | O(n^2) | O(log n) | No |
| 简单选择 | O(n^2) | O(n^2) | O(n^2) | O(1) | No |
| 堆 | O(n log n) | O(n log n) | O(n log n) | O(1) | No |
| 归并 | O(n log n) | O(n log n) | O(n log n) | O(n) | Yes |
| 基数 | O(d(n+r)) | O(d(n+r)) | O(d(n+r)) | O(r) | Yes |

> 注意：希尔最坏O(n^2)是原始Shell增量序列的界，优化序列可达O(n log^2 n)。O(n^1.3)是经验值。归并和基数属于**内部排序**。

### 10.2 新增算法（Drozdek 9）

梳排序（冒泡改进，gap>1比较）、计数排序（O(n+k)，非比较，稳定）。

### 10.3 排序选择

基本有序->插入 | 随机量大->快速 | 需最坏保证->堆/归并 | 需稳定->归并 | Top-K->堆 | 外存->多路归并 | 小范围->计数 | 定长键->基数

### 10.4 算法特征

| 特征 | 哪些算法 |
|------|---------|
| 比较次数与初始无关 | 简单选择、归并、折半插入 |
| 每趟确定最终位置 | 冒泡(尾)、选择(头)、快排(枢轴) |
| 最后一趟前可能全不在位 | 直接插入 |

### 10.5 比较排序下界

基于决策树模型：ceil(log2(n!)) = Omega(n log n)。计数排序和基数排序绕过此界（利用键结构而不比较）。

### 10.6 外部排序（Shaffer 第8章）

归并段(Run)：一次内存可排序的数据块。置换-选择排序：生成比内存更大的归并段。多路归并：同时归并k个run。败者树：O(log k)选择k路最小值。趟数 s = ceil(log_k m)。

## 第11章 查找与索引

### 11.1 查找算法

| 方法 | 前提 | ASL成功 | ASL失败 |
|------|------|---------|---------|
| 顺序查找 | 无 | (n+1)/2 | n+1 |
| 折半查找 | 有序+随机访问 | ~log2(n+1)-1 | ~log2(n+1) |
| 分块查找 | 块间有序 | sqrt(n)+1 | — |

二分查找用半开区间[lo,hi)减少±1错误。lower_bound找第一个>=x。

### 11.2 自组织列表（Shaffer 9.2 + Drozdek 3.5）

移到前端法、换位法、计数法。适合80/20偏斜访问。

### 11.3 索引结构

ISAM（Shaffer 10.2）：静态多级索引，早期数据库。2-3树（Shaffer 10.4）：最简单的B树变体。

### 11.4 Trie家族（Drozdek 7.2/13.1 + Morin 13）

| 结构 | 查找 | 空间 | 用途 |
|------|:--:|:--:|------|
| Trie（前缀树） | O(key_len) | O(n*key_len) | 字典/自动补全 |
| BinaryTrie | O(w) | O(n) | 整数集 |
| XFastTrie | O(1)期望 | O(nw) | 整数集超快查找 |
| YFastTrie | O(log w) | O(n) | 整数集空间优化 |
| 后缀Trie | — | O(n^2) | 子串查找 |
| 后缀数组 | O(log n) | O(n) | 紧凑替代后缀树 |

### 11.5 位向量（Shaffer 9.3）

单个bit表示元素存在，适合全集较小且密集的场景。

## 第12章 递归与算法设计

### 12.1 递归的类型（Drozdek 5）

尾递归（可优化为循环）、非尾递归、间接递归、嵌套递归。本质相同——每次调用必须严格趋近基例。

### 12.2 递归本质（Erickson 1）

递归 = 归约（reduction）：把大问题归约为同类的小问题。回溯 = 递归 + 撤销选择。

### 12.3 动态规划（Erickson 3）

区分：回溯（暴力+剪枝）-> 记忆化（回溯+缓存 = 自顶向下DP）-> DP（自底向上填表）。

DP本质：Smart recursion——先确定递归结构，再决定记忆化还是填表。

### 12.4 贪心（Erickson 4）

贪心取当前最优，不保证全局最优。哈夫曼编码和Gale-Shapley稳定婚姻是成功的贪心例子。

## 第13章 NP-完全性

### 13.1 基本定义

- **P**：可快速求解的问题
- **NP**：可快速验证的问题
- **NP-完全**：NP中最难的一类，若一个被多项式解决则全部被解决

### 13.2 归约链（Erickson 第12章）

CircuitSAT -> 3SAT -> 最大独立集 -> 团/顶点覆盖 -> 哈密顿回路 -> 子集和

| 归约 | 核心技巧 |
|------|---------|
| CircuitSAT->3SAT | 变量模拟逻辑门 |
| 3SAT->独立集 | 每个子句建三角形 |
| 独立集->团 | 图的补图 |
| 独立集->顶点覆盖 | 补集关系 |
| 顶点覆盖->哈密顿回路 | "选择器"小工具 |
| 顶点覆盖->子集和 | 数字编码图结构 |

## 第14章 进阶专题

### 14.1 独特数据结构（Morin 2-3章）

**接口体系**：USet（无序集合）、SSet（有序集合）、List（线性序列）、Queue、Stack、Deque。选择结构时应先明确所需接口。

**DualArrayDeque**：两个ArrayStack背靠背。失衡时balance()均匀分配。摊还O(1)。

**RootishArrayStack**：边长递增（第k个长k）的数组组。空间浪费O(sqrt n)而非O(n)，保持O(1)随机访问。

**SEList**：双链表节点内含小数组（块~sqrt n）。查找O(sqrt n)。平衡纯数组与纯链表的折中。

### 14.2 空间数据结构（Drozdek 6.11/7.1.5-7）

**k-d树**：k维空间点查询，每层交替用不同维度切分。**R-树**：空间矩形索引，GIS数据库。**四叉树**：二维递归四分。

### 14.3 设计模式（Shaffer 1.3）

Flyweight（享元）共享只读节点。Visitor（访问者）分离遍历与操作。Composite（组合）树结构天然组合。Strategy（策略）切换算法。Iterator（迭代器）统一遍历接口。

### 14.4 数据压缩（Drozdek 11）

哈夫曼编码、自适应哈夫曼（动态调整频率）、游程编码（连续重复字符）、LZW（字典映射，GIF/Unix使用）。

### 14.5 内存管理（Drozdek 12）

**伙伴系统**：2的幂分配，相邻伙伴合并。快速合并但内部碎片。

**垃圾回收**：标记-清除（可达标记+清除不可达）、复制（从根复制活对象）、增量（穿插执行减少暂停）、分代（按年龄分区，年轻代频回收）。

### 14.6 字符串匹配（Drozdek 13）

| 算法 | 复杂度 | 特点 |
|------|:--:|------|
| BF | O(nm) | 简单 |
| KMP | O(n+m) | 主串不回溯 |
| Boyer-Moore | O(n/m)最好 | 从右匹配 |
| Rabin-Karp | O(n+m)期望 | 滚动哈希 |
| 编辑距离 | O(nm) | DP |

## 附录A：公式速查卡

### 树
- n0 = n2 + 1（任意二叉树）
- 空指针数 = n + 1（二叉链表）
- 线索数 = n + 1
- 哈夫曼总节点 = 2*n0 - 1
- 满二叉树定理：叶数 = 内部节点数 + 1

### 图
- 无向完全边 = n(n-1)/2
- 有向完全边 = n(n-1)
- 连通最少 = n-1（无向）
- 强连通最少 = n（有向）
- 生成树 = n-1边

### 卡特兰数
- C_n = C(2n,n)/(n+1)
- C_3=5, C_4=14, C_5=42

### 外部排序
- 趟数 s = ceil(log_k m)
- 败者树 O(log k)

### 红黑树
- 高度 <= 2log2(n+1)
- 五性质：二色、根黑、叶黑、无连续红、黑高等

### 并查集
- 路径压缩+按秩合并：O(m * alpha(n))
- alpha(n) <= 4（实践中）

### Johnson算法
- O(V^2 log V + VE)
- 三步：势能+重赋权+Dijkstra
